# 启动流程设计

> 决定日期: 2026-05-21
> 相关: `design-docs/multi-instance.md`

## 流程

```
应用启动
  │
  ├─ 检查本地是否有已保存的实例配置
  │   └─ Electron 主进程文件存储中保存的实例索引 + 凭证摘要
  │
  ├─ [有已保存实例] → 尝试连接 → 
  │   ├─ 成功 → 进入主界面，加载该实例
  │   └─ 失败 → 进入「连接向导」，可修改/重试配置
  │
  └─ [无已保存实例] → 进入「连接向导」（独立窗口/全屏遮罩）
                        │
                        ├─ 自动发现 → 扫描本地服务（正在运行的 OpenClaw 实例）
                        │   ├─ 发现一个 → 自动连接 → 设为当前实例 → 进入主界面
                        │   └─ 发现多个 → 展示列表让用户选择
                        │
                        ├─ 手动输入 → 填写 URL + Token / API Key → 测试连接
                        │   └─ 成功 → 设为当前实例 → 进入主界面
                        │
                        └─ 连接成功后，保存配置到本地实例列表
```

## 核心逻辑

| 步骤 | 条件 | 行为 |
|------|------|------|
| 1 | 页面加载 | 读取本地存储中的实例配置列表 |
| 2 | 有「当前实例」配置 | 尝试连接 OpenClaw Gateway |
| 3 | 连接成功 | 进入主界面，实例状态=在线 |
| 4 | 连接失败 / 无配置 | 弹出「连接向导」| |
| 5 | 连接向导 → 自动发现 | 扫描本地运行的 OpenClaw 服务 |
| 6 | 发现单个实例 | 自动连接 → 保存 → 进入主界面 |
| 7 | 发现多个实例 | 展示列表，用户选择 → 连接 → 进入主界面 |
| 8 | 未发现 / 用户选择手动 | 弹出手动输入表单 → 测试连接 → 保存 → 进入主界面 |
| 9 | 所有情况 | 连接成功后自动保存到本地实例列表，标记为当前实例 |

## 连接方式

1. **自动发现**: 扫描本地运行的 OpenClaw 服务（如扫描 localhost 端口、解析 openclaw-cli 输出）
2. **手动输入**: URL + Token / API Key

## 断线策略

- **自动行为**: 静默自动重连（后台持续尝试）
- **UI 反馈**: 消息条、状态栏图标、侧边栏实例状态标识
- 不阻断用户操作，保持当前页面可见
- 重连成功后自动恢复

## 本地存储

- 采用 `docs/design-docs/local-instance-storage.md` 定义的响应式内存层 + Electron 文件后端。
- 启动时先 hydrate Zustand store；运行中先更新内存状态，再异步持久化。
- 旧 `localStorage` 数据仅作为一次性迁移来源，迁移后应清理。
