## 截图追溯 P0 基线

2026-06-28，用户再次贴出前序复审截图并指出：“这里面的内容 你是不是已经完全丢失 忘记了，我觉得这些也是 P0 级别的内容”。因此下面内容是当前 P0 规划和验收来源，不得在后续推进中降级为 P1/P2，也不得因为先推进 Artifact、ActionRun 或单个页面切片而被弱化。

- **开箱体验与工作系统金线**：Gateway 不能被隐藏，但必须从“用户目标”降级为“基础设施”；第一感受应是“创建我的工作系统”，不是“连接控制台”。
- **自由边界**：保持自由、不被第三方商业生态绑定是 P0 产品约束；外部生态入口可以暂不作为当前代码实施阻塞，但不能把“自由”从 P0 验收中移走。
- **Dashboard 真实推进状态**：首屏应优先回答今日继续、待确认、卡住事项、最近成果、知识动态和本周新增成果；Gateway 健康状态只是基础设施状态。
- **Repository 初始化与普通人语言包**：首次连接成功后应引导创建本地工作仓库；UI 第一层优先使用资料、知识、事项、计划、执行记录、成果、复盘。
- **Knowledge 导入/消化/健康检查**：导入中心、未消化资料队列、AI 消化计划、写入 `wiki/` / `wiki/index.md` / `wiki/log.md`、健康检查和 `reviews/weekly/` 归档都属于 P0 闭环。
- **事务推进闭环**：工作事项要有唯一 ID，ActionRun 要归属事项，执行记录要写回事项页，结束后要触发更新状态、沉淀成果、更新知识库、写入复盘等尾动作。
- **开始一件事闭环**：用户一句话 -> 事项 -> 计划 -> 执行 -> 产物 -> 复盘是专题级 P0，不是体验优化。
- **可复用资产一等对象**：工具、脚本、模板、工作流、提示词、检查清单等需要来源、版本、权限、审批边界、运行记录和产出关联。

说明：外部生态入口（SkillHub / ClawHub / GitHub release 等）当前不作为本阶段具体代码实施阻塞；这不等于自由边界不是 P0。P0 验收仍必须持续确认用户可通过开源源码、协议开放、数据可迁移和仓库事实源保持自由，不被任何第三方商业生态绑定。
