## 启动流程 (Bootstrap)

### 流程概览

```
应用启动
  │
  ├─ 检查本地是否有已保存的实例配置
  │   └─ (localStorage / IndexedDB 中保存的 gatewayUrl + token)
  │
  ├─ [有已保存实例] → 尝试连接 → 
  │   ├─ 成功 → 进入主界面，加载该实例
  │   └─ 失败 → 进入「连接向导」，可修改/重试配置
  │
  └─ [无已保存实例] → 进入「连接向导」（独立窗口/全屏遮罩）
                        │
                        ├─ 自动发现 → 扫描本地服务（正在运行的 OpenClaw 实例）
                        │   ├─ 发现一个 → 自动连接 → 设为当前实例 → 进入主界面
                        │   └─ 发现多个 → 展示列表让用户选择
                        │
                        ├─ 手动输入 → 填写 URL + Token / API Key → 测试连接
                        │   └─ 成功 → 设为当前实例 → 进入主界面
                        │
                        └─ 连接成功后，保存配置到本地实例列表
```

### 核心逻辑

| 步骤 | 条件 | 行为 |
|------|------|------|
| 1 | 页面加载 | 读取本地存储中的实例配置列表 |
| 2 | 有「当前实例」配置 | 尝试连接 OpenClaw Gateway |
| 3 | 连接成功 | 进入主界面，实例状态=在线 |
| 4 | 连接失败 / 无配置 | 弹出「连接向导」| |
| 5 | 连接向导 → 自动发现 | 扫描本地运行的 OpenClaw 服务 |
| 6 | 发现单个实例 | 自动连接 → 保存 → 进入主界面 |
| 7 | 发现多个实例 | 展示列表，用户选择 → 连接 → 进入主界面 |
| 8 | 未发现 / 用户选择手动 | 弹出手动输入表单 → 测试连接 → 保存 → 进入主界面 |
| 9 | 所有情况 | 连接成功后自动保存到本地实例列表，标记为当前实例 |

### 实例管理

- 支持保存多个实例配置（historical connections）
- 有且仅有一个「当前活跃实例」
- 实例配置存储在本地（浏览器 localStorage / IndexedDB）
- 连接本质上是一个 Gateway WebSocket / HTTP 端点

### 多实例管理

- 支持保存多个实例连接配置
- 通过标签页（Tabs）的方式管理与切换
- 标签页展示：实例名称、连接状态（在线/离线）、当前活跃标示
- 实例配置存储在浏览器本地
- 标签页可增删改名

### 断线策略

- **自动行为**: 静默自动重连（后台持续尝试）
- **UI 反馈**: 界面提示（消息条、状态栏图标、侧边栏实例状态标识）
- 不阻断用户操作，保持当前页面内容可见
- 重连成功后自动恢复，移除断线提示

## 模块详述

### 1. Dashboard（首页）

数据来源: OpenClaw 自身拉取 + 自增功能 (TODO/看板)

布局优先级:

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | 快捷对话框 | 大输入框，直接唤起新会话，最常用入口 |
| P1 | 今日概览卡片 | 活跃 Agent 数、待执行任务数、今日会话数、运行中工作流 |
| P2 | 活跃看板项 | 当前正在执行或最近完成的看板卡片（最近 5 条） |
| P3 | Agent Teams 列表 | 精简的团队卡片，点击跳到完整页面 |
| P4 | TODO List | 待办列表（轻量） |

TODO List = 自增功能，OpenClaw 没有自带。

### 2. 新会话

独立页面，新建一个 AI 会话。

### 3. 当前会话

独立页面，展示与 AI 的对话内容。

### 4. 搜索

独立页面，支持两种搜索：
- 会话搜索（历史对话检索）
- 网络搜索

### 5. 工作区

Workspace 文件管理。

### 6. 定时任务

Cron Jobs 管理界面。

### 7. 看板 (Kanban) — ★ 核心差异化功能

**概念**: 事件驱动的 Agent 工作流编排，非定时性质。
**参考**: [Hermes Agent Kanban](https://hermes-agent.nousresearch.com/docs/user-guide/features/kanban)

**核心设计**:
- 每一项卡片 = 一个工作内容/任务包
- 有一主 Agent 负责：拆解任务 → 编码 → 拉起子 Agent 执行
- 事件驱动流转，非定时触发

**状态流转（初步）**:

```
┌──────┐   →   ┌──────┐   →   ┌──────┐   →   ┌──────┐
│ 待办  │       │ 执行中 │       │ 审核  │       │ 完成  │
│(Pending)│     │(Running)│     │(Review)│     │(Done) │
└──────┘       └──────┘       └──────┘       └──────┘
                      │              │
                      ↓              ↓
                  ┌──────┐       ┌──────┐
                  │ 阻塞  │       │ 失败  │
                  │(Blocked)│    │(Failed)│
                  └──────┘       └──────┘
```

**卡片展开详情**:
- 主 Agent 当前状态（编码中 / 等待子 Agent / 等待审核）
- 子 Agent 调用链（树形展开）
- 执行日志流（实时）
- 手动介入按钮（修改 / 重试 / 跳过）

**与 Dashboard 的关系**: 看板上的活跃项在首页 P2 区域展示摘要。

### 8. 团队 (Agent Teams)

OpenClaw 代理的平面展示与编辑。以列表/卡片形式展示所有注册的 Agent，支持编辑配置。

### 9. 3D虚拟办公室 — ★ 创新功能

**概念**: 把每个 Agent 具象化为 3D 空间中的角色/工位/房间。

**Agent 状态视觉表现**:

| Agent 状态 | 3D 表现 |
|-----------|---------|
| 空闲 | 角色坐在工位前，灯光白色 |
| 工作中 | 工位屏幕闪烁，角色身体微微前倾 |
| 忙碌 | 橙色/红色光环 |
| 离线 | 工位灰色，角色半透明或消失 |
| 协作中 | 两个 Agent 之间有连线/光缆流动 |

**协作关系墙**: 在办公室的一面墙壁上渲染 Agent 之间的协作关系：
- 网络拓扑图 / 热力图 / 时间线
- 展示哪些 Agent 之间最近有过协作（频率、时长、任务类型）

### 10. 扩展

统一管理界面，分为两种类型：
- **技能 (Skills)**: OpenClaw 的 SKILL.md 体系，能力描述文件
- **插件 (Plugins)**: MCP / Tool 接入，底层执行单元

### 11. 记忆

Memory 管理界面，查看/编辑长期记忆。

### 12. 设置等

系统配置，用户偏好，API 密钥管理等。
